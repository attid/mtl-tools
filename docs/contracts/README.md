# Contracts Registry

Этот раздел хранит спецификации публичных текстовых контрактов, которые бот принимает/отправляет в каналах.

## Назначение
- Зафиксировать форматы сообщений как source of truth.
- Синхронизировать реализацию, тесты и документацию.
- Упростить безопасные изменения через versioning/deprecation.

## Текущие контракты
- `skynet-mmwb-ping-pong.md` — heartbeat контракт между SkyNet и MMWB.
- `skynet-helper-taken-closed.md` — helper-события taken/closed + ACK.

## Правила поддержки
- Изменение публичного контракта требует обновления соответствующего файла в `docs/contracts/` в том же PR.
- Любой новый кейс в контракте должен подтверждаться тестом (минимум unit/router-level).
- Формат примеров обязателен: для каждого контракта должны быть valid и invalid примеры.

## Формат спецификации
- **Direction**: кто отправляет, кто обрабатывает.
- **Pattern**: обязательные теги/ключи и требования к порядку.
- **Fields**: обязательные и опциональные поля.
- **ACK/Errors**: ожидаемые ответы и условия игнора.
- **Examples**: минимум 2 valid и 2 invalid примера.
