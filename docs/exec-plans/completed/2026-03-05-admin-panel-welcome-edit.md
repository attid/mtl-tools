# ADMIN-PANEL-WELCOME-EDIT: Исправить UX редактирования приветствия в /admin

## Контекст
Сейчас в `/admin` в блоке приветствия значения отображаются урезанно, а при нажатии редактирования пользователь не видит текущий текст, что делает изменение неудобным и рискованным.

Ссылка: `routers/admin_panel.py` (`/admin` + `Welcome Settings`).

## План изменений
1. [x] В `routers/admin_panel.py` расширить `cb_show_welcome_settings` для более удобного отображения текущего текста и кнопки приветствия без жёсткого обрезания.
2. [x] В `cb_edit_welcome` добавить в prompt текущие сохранённые значения (текст/кнопку), чтобы пользователь видел, что редактирует.
3. [x] После сохранения в `process_welcome_message` и `process_welcome_button` вернуть пользователя в экран `Welcome Settings` с актуальными значениями, а не только подтверждением отдельным сообщением.
4. [x] Проверить соответствие типам и импорту в `routers/admin_panel.py` (при необходимости добавить защиту от `None`).
5. [ ] Обновить/добавить тесты для сценариев `cb_show_welcome_settings`, `cb_edit_welcome`, сохранения/возврата настроек, если есть подходящий существующий тестовый каркас.
6. [x] Обновить документацию в `docs/` только при необходимости (если меняется поведение пользовательского интерфейса, зафиксировать в заметке или changelog, если практикуется).
7. [ ] Проверка: `just lint` и целевые unit-тесты для изменённого роутера (или ближайший эквивалент команд проекта).

## Риски и открытые вопросы
- Риск: формат отображения может стать слишком длинным для Telegram-сообщений без дополнительного лимита.
- Риск: изменение навигации FSM после сохранения может повлиять на пользователские сценарии в редком случае.
- Вопрос: достаточно ли unit-тестов для текущего роутера или требуется добавить новый файл тестов под `admin_panel`.

## Верификация
- Шаг 1: открыть `/admin` в приватном чате, выбрать чат и войти в `Welcome Settings`.
- Шаг 2: проверить, что текущий текст/кнопка видны в читаемом формате и корректно эскейпятся для HTML.
- Шаг 3: нажать `Edit Message`/`Edit Button`, убедиться, что в prompt видно текущее значение.
- Шаг 4: ввести новое значение и убедиться, что экран возвращается в `Welcome Settings` с обновлёнными данными.
- Артефакты: валидация сообщений в чате `/admin` + корректный `inline_markup` у экрана настроек приветствия.
