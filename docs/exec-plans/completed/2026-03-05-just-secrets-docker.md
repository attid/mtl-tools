# JUST-SECRETS-DOCKER: Запуск gitleaks через Docker в команде just secrets

## Контекст
Команда `just secrets` сейчас зависит от локально установленного `gitleaks`, которого может не быть в окружении.
Нужно сделать её запускаемой в средах без локальной установки через `docker run`.

## План изменений
1. [x] Обновить рецепт `secrets` в `justfile`.
2. [x] Ввести fallback: если `gitleaks` есть локально — использовать его, иначе запускать через `docker run`.
3. [x] Протестировать `just secrets` в текущем окружении на этапе проверки (факт отсутствия локального бинаря должен корректно переходить к Docker).
4. [x] Обновить план выполнения после завершения и переместить в `docs/exec-plans/completed/`.

## Риски и открытые вопросы
- Риск: docker отсутствует на CI/локальной машине, тогда команда завершится с ошибкой, как и было раньше.
- Риск: поведение `-v` может отличаться для очень старых версий image `zricethezav/gitleaks`.

## Верификация
- Выполнить `just secrets`.
- Ожидаемый результат: команда отрабатывает без локального `gitleaks` и возвращает результат сканирования.
