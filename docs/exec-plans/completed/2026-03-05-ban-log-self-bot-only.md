# ban-log-self-bot-only: дедуп только для follow-up от собственного бота

## Контекст
- Предыдущее условие `event.from_user.is_bot` слишком широкое и подавляет логи для банов от других ботов.
- Нужно подавлять только follow-up `KICKED`, который инициирован именно текущим ботом после собственного auto-ban.

## План изменений
1. [x] Добавить регрессионный тест: follow-up `KICKED` от `router_app_context.bot.id` не создаёт второй `manual`-лог.
2. [x] Добавить регрессионный тест: `KICKED` от другого бота по-прежнему логируется как manual-ban.
3. [x] Обновить `routers/welcome.py`, чтобы дедуп срабатывал только для собственного бота.
4. [x] Обновить/добавить тесты.
5. [x] Проверка: `uv run pytest tests/routers/test_welcome.py -q`.

## Риски и открытые вопросы
- Нельзя снова получить ложный дубль для auto-ban от `SkyNet`.
- Нельзя потерять лог при бане, инициированном сторонним ботом-админом.

## Верификация
- Воспроизвести auto-ban с follow-up `KICKED` от собственного бота и убедиться, что лог один.
- Воспроизвести `KICKED` от чужого бота и убедиться, что `manual`-лог сохраняется.
