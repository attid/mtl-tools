# 2026-03-18-fix-reaction-admin-check: Fix "Local admins not set yet" for non-moderation reactions

## Контекст
- Пользователи получают сообщение "Local admins not set yet" при использовании любых кастомных эмодзи в топиках, даже если эмодзи не являются командами модерации (мьют, удаление).
- Бот сейчас проверяет наличие администраторов топика сразу, как только видит кастомный эмодзи, не убедившись, что этот эмодзи является командой.

## План изменений
1. [x] Проанализировать `routers/admin_core.py`, функцию `message_reaction`.
2. [x] Перенести проверку `reaction.custom_emoji_id` (на принадлежность к командам мьюта или удаления) *до* проверки прав администратора.
3. [x] Убедиться, что если эмодзи не входит в список команд модерации, функция завершается без отправки сообщений.
4. [x] Обновить существующие тесты в `tests/routers/test_admin_core.py`, чтобы подтвердить новое поведение.
5. [x] Проверка: `just lint`, `just format`, `just types`, `just test`.

## Риски и открытые вопросы
- Нет рисков. Изменение только сужает круг случаев, когда бот отправляет сообщения.

## Верификация
- Тест `test_message_reaction_no_action` (или аналогичный) должен подтверждать, что на неизвестный кастомный эмодзи бот не отвечает.
- Команды мьюта и удаления по-прежнему должны требовать настройки локальных админов.
