# AI-FIRST-100PCT: Программа перехода репозитория на AI-First

## Контекст
Цель: перевести проект в режим AI-first на 100% без "большого взрыва", сохраняя рабочее состояние после каждой итерации.

Ограничение: каждая итерация должна завершаться в состоянии "бот запускается, критичные команды работают, тесты/проверки итерации зелёные".

## Целевая модель (Definition of Done программы)
1. Все нетривиальные задачи выполняются через execution plan в `docs/exec-plans/`.
2. Контракты событий/API документированы и проверяются тестами.
3. Архитектурные границы формально проверяются линтерами/структурными тестами.
4. Стандартный инженерный цикл сводится к одной команде (`just check` или эквивалент) и обязателен перед merge.
5. Наблюдаемость достаточна для автономной диагностики агентом.
6. Документация живёт рядом с кодом и обновляется вместе с изменениями.

## Стратегия миграции
Подход: phased migration (expand -> migrate -> enforce -> cleanup).

Правило безопасности: сначала добавляем новое правило/инструмент в "advisory mode", затем переводим в "required mode".

## Инварианты каждой итерации
1. Нет массовых рефакторингов без необходимости.
2. Нет отключения существующих тестов/проверок.
3. Есть явный rollback-путь (revert коммита/флага).
4. Есть короткая верификация работоспособности.

## Метрики прогресса
1. `% tasks with execution plan`.
2. `% public contracts with explicit docs + tests`.
3. `% modules covered by architectural guardrails`.
4. `just check` success rate on main.
5. `% runbooks for top incidents`.
6. Количество "manual tribal decisions" (должно снижаться).

## Верификация
1. После каждой итерации: запуск check-набора итерации и smoke сценария запуска бота.
2. Каждые 2-3 итерации: срез метрик прогресса и корректировка backlog.
3. Каждые 5 итераций: мини-аудит соответствия AI_FIRST.md.
4. Финальная проверка: все пункты целевой модели закрыты, проект стабилен.
